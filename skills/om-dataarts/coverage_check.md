# 增量覆盖率检测 Skill

## 触发条件
- 用户请求检测增量覆盖率
- 用户请求检查测试覆盖率是否达标
- 用户提到 "增量覆盖率"、"UT覆盖率"、"测试覆盖率"

## 检测步骤

### 1. 获取变更文件列表
```bash
git diff --name-only origin/main...HEAD | grep -v "^tests/"
```

### 2. 运行测试并生成覆盖率报告
```bash
pytest --cov=./ --cov-report=xml:coverage.xml --cov-report=term-missing
```

### 3. 计算增量覆盖率
```bash
# 安装 diff-cover
pip install diff-cover

# 生成 patch
git diff origin/main...HEAD > pr_changes.patch

# 计算增量覆盖率
diff-cover --diff-file pr_changes.patch coverage.xml --exclude "test_*.py"
```

### 4. 分析缺失覆盖的代码行
查看报告中 "Missing lines" 部分，定位未覆盖的新增代码。

### 5. 生成覆盖率报告
输出格式：
| 文件 | 覆盖率 | 缺失行号 |
|------|--------|----------|
| ... | ...% | 行号列表 |

### 6. 判断是否达标
- 阈值：80%
- 覆盖率 ≥ 80%：达标
- 覆盖率 < 80%：未达标，需要补充测试用例

## 常见问题解决

### 问题1：测试用例未覆盖新增代码
**原因**：测试用例独立测试逻辑，未调用源代码方法
**解决**：修改测试用例，调用源代码中的方法

### 问题2：异常分支未覆盖
**原因**：正常测试流程不会触发异常
**解决**：使用 Mock 或构造特殊输入触发异常分支

### 问题3：复杂依赖导致无法调用
**原因**：源代码依赖外部服务（数据库、API等）
**解决**：使用 Mock 替换外部依赖
